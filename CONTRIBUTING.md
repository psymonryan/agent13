# Contributing

Contributions are welcome! But not in the traditional PR sense.

## The Workflow

1. Found a bug or have a feature idea? → **Open a GitHub issue**
2. Describe what you want clearly (what problem does it solve?)
3. I'll review the issue and, when ready, use it as input to the agent
4. The agent implements it, I review the result, and if it's good → it ships

## Why Not Pull Requests?

This project is developed with AI assistance. Merging external PRs creates merge conflicts, requires careful review of unfamiliar code, and disrupts the single-developer workflow. Issues work better here.

## What I'm Looking For

- **Clear bug reports** - what did you expect vs what happened?
- **Feature requests with a use case** - why would this be useful?
- **Real-world usage feedback** - how are you using agent13? What could be better?

## Getting Started

1. Check existing issues - your idea might already be there
2. Open a new issue with a clear title and description
3. Say hi - I'm happy to hear from you

Looking forward to hearing your ideas.
